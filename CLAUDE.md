# YouTube → Notion 콘텐츠 처리 Instruction

## 기본 규칙
- 영어 콘텐츠는 한글 번역 후 정제 (한글오타가 최근에 너무 많아! 오타발생하지 않도록 각별히 주의할 것)
- 한글 콘텐츠는 그대로 정제하여 일목요연하게 정리
- 1000줄 이내 (노션 저장 호환), 내용 적으면 채울 필요 없음 (중복되지 않도록 작성할 것)
- rating 속성은 건드리지 않음 (사용자가 직접 설정)
- 도구 승인은 항상 YES, 묻지 말 것

### 콘텐츠 유형별 글쓰기 전략

영상 성격에 따라 아래 2가지 중 적합한 스타일 선택할 것

**A. 스토리텔링형** (인물 인터뷰, 창업기, 실패담 등 서사가 있는 콘텐츠)
- 책을 출판할때 하나의 챕터를 구성한다는 생각으로 작성할 것
- 가독성 좋게 구성해야 하고, 스토리텔링형으로 구성할 것
- 짧고 펀치 있는 문장으로 리듬 만들기 ("그런데 완전히 망쳤다.")
- 화자의 목소리를 살려 1인칭/3인칭 서사로 전달
- 핵심 깨달음은 **bold**로 강조
- 스타일 참고: "AI 에이전트 판매로 월 3억 버는 것은 불가능하다"

**B. 정보 전달형** (뉴스, 트렌드, 방법론 등 정보가 중심인 콘텐츠)
- 핵심 메시지를 먼저 제시하고 근거를 뒷받침하는 구조
- 전문 용어는 쉽게 풀어서 설명
- 데이터/통계는 시각적으로 보이게 bullet point 활용
- 스타일 참고: "2026년 AI 트렌드 5가지"

### 공통 규칙
- 정보 나열이 아니라 '이야기 흐름'이 느껴지게 구성할 것
- "무슨 일이 있었는지 → 왜 중요한지 → 그래서 어떤 변화가 생기는지" 흐름 유지
- 과장된 표현이나 유튜브식 말투는 피하고, 차분한 해설 톤으로 작성
- 하나의 문단은 3~5줄을 넘기지 말 것
- 문단 → bullet → 문단 흐름을 섞어서 읽는 리듬을 만들 것
- callout과 quote는 남용하지 말 것 (섹션당 1~2개)
- 원본에 있는 구체적 수치, 금액, 사례명은 반드시 보존할 것

### 제목 작성
- 원본 제목을 그대로 쓰지 말 것. 핵심 메시지를 담아 한글로 재작성
- 좋은 예: "1억 달러 SaaS 카피해서 월 1,500만원 버는 전략", "혼자서 연 17억 버는 디자이너의 비밀"
- 나쁜 예: video_id 그대로, 영어 제목 그대로, 너무 긴 제목

### 출력 구조
1. 도입부: 한 문단 (왜 이 이야기를 읽어야 하는지, 독자의 관심을 끄는 훅)
2. 본문: 스토리/분석 (영상의 핵심을 '읽을 가치 있는 글'로 재구성)
3. 마무리: 짧은 정리 문단 (이 변화가 의미하는 것, 액션 아이템)

### 목표
노션에 정리했을 때 '요약본'이 아니라 '읽을 가치가 있는 글'처럼 느껴지게 할 것.


## 작업 순서
1. Notion "Agent References" (DB ID: `28ab5922-0286-804c-8b6f-e9950825fb86`) 에 페이지 생성 (제목만)
2. **YouTube URL이 있으면 페이지 생성 직후, 본문 콘텐츠를 추가하기 전에 아래 작업을 먼저 수행할 것:**
   - 첫 번째 블록: type="image"인 블록으로 `https://img.youtube.com/vi/{VIDEO_ID}/maxresdefault.jpg` 이미지를 추가 (embed나 bookmark이 아님! 반드시 image 블록)
   - 두 번째 블록: YouTube 원본 URL 텍스트 링크
   - 이 두 블록이 추가된 후에 본문 콘텐츠를 이어서 작성할 것
   - **절대 cover에만 넣고 본문에 이미지를 빠뜨리지 말 것**

3. 정제된 콘텐츠를 섹션별로 업로드 (실패 시 해당 섹션만 재시도)
4. Cover/Icon 설정 → 이모티콘은 모두 제외할 것

## Cover/Icon 우선순위

| 우선순위 | 조건 | 처리 |
|---------|------|------|
| 1 | YouTube 비디오 있음 | 썸네일 → Cover |
| 2 | 이미지 블록 있음 | 첫 번째 이미지 → Cover |
| 3 | 링크/북마크만 있음 | OG 이미지 |
| 4 | 콘텐츠 없음 | 제목 기반 |

## 주의사항
- Cover는 Notion MCP 도구로 직접 설정할 것 (API 키 없다는 핑계 금지)
- 콘텐츠가 길면 섹션별 분할 업로드, 전체 재작업 금지
- 긴 콘텐츠를 한 번에 전송 시도 금지
- 승인 여부 물어보기 금지
